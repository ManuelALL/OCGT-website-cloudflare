# OCGT Website — Architecture

## Stack
- **Static HTML** (no framework, no runtime build) — single source: `OCGT_website.html`
- **Python prerender** → 20 SEO-optimized static pages in `dist/`
- **Cloudflare Worker with Static Assets** (`worker/index.js` + `[assets]` binding in `wrangler.toml`) for global edge delivery
- The same Worker handles the contact form (`POST /api/contact` → Brevo API). `functions/api/contact.js` is a legacy Cloudflare Pages Function kept for reference from an earlier Pages-based setup — the live path is `worker/index.js`.

## File map

```
OCGT_website.html        Master SPA. Single source of truth for HTML/CSS/JS/META.
build-prerender.py       Generates dist/ — one HTML per route, plus _headers,
                         _redirects, service worker registration, image lazy-loading.
worker/index.js          The live Worker: serves dist/ via [assets], handles POST /api/contact via Brevo.
functions/api/contact.js Legacy Cloudflare Pages Function (kept for reference, not the live path).
wrangler.toml            Cloudflare Worker (Static Assets) config — name, main, [assets] binding.
manifest.webmanifest     PWA manifest (icons, theme color, install prompt).
sitemap.xml / robots.txt SEO crawl hints.

Images/                  Hero + section imagery (AVIF/WebP/JPG fallbacks).
logos/, icons/           Brand + UI graphics.
marketing/               Press kit, case-study assets.
04 Videos/               Background and section video.

dist/                    Build output. Never edit by hand. Regenerate with:
                           python3 build-prerender.py
```

## Routing

The SPA defines 20 routes in the `ROUTES` map at the top of `build-prerender.py`. Each route maps a URL slug → page-id (e.g. `geotechnik` → `geotechnik`, `bauueberwachung` → `a1`). The build:

1. Reads master HTML.
2. For each route, clones the HTML, marks `<div id="p-{pageId}">` as `class="page on"` so it renders without JS.
3. Bakes the route's `<title>`, `<meta name="description">`, `<link rel="canonical">`, og:title/url/desc into the head.
4. Writes to `dist/{slug}/index.html`.

Result: Google indexes 20 distinct pages with unique meta, while client-side the SPA still works for in-session navigation.

## Per-route SEO metadata

Source: the `META = { ... }` object inside `OCGT_website.html`. Each entry has DE + EN title and description. The build script extracts this with a regex; if you change the META structure, update `extract_meta_map()` in `build-prerender.py`.

## Cloudflare deployment

- `_headers` — security headers (HSTS, X-Frame-Options, Referrer-Policy, Permissions-Policy) + immutable cache for `/Images/*`, `/logos/*`, etc. + short cache for `*.html`.
- No `_redirects` file — Workers Static Assets doesn't use it (`build-prerender.py` deletes any stale one from an earlier Pages build). `ocgt.de` (apex) → `www.ocgt.de` is a Cloudflare **Redirect Rule** (dashboard-only, not in this repo). Unknown paths get a real `404` via `wrangler.toml`'s `not_found_handling = "404-page"` + the generated `dist/404.html`.
- `worker/index.js` — runs at the edge, serves `dist/` via the `[assets]` binding and handles `POST /api/contact`. Required env vars/secrets: `BREVO_API_KEY`, `CONTACT_TO_EMAIL`, `CONTACT_FROM_EMAIL`, `CONTACT_FROM_NAME`, optionally `TURNSTILE_SECRET_KEY`.

Deploy:
```bash
python3 build-prerender.py
wrangler deploy
```

## Performance optimizations (applied by build)

| Optimization | Location | Effect |
|---|---|---|
| Lazy-load images below the fold | build-prerender.py post-pass | Smaller initial payload, faster LCP |
| `decoding="async"` on all `<img>` | build-prerender.py post-pass | Non-blocking image decode |
| Hero asset preload | injected into `<head>` | First image shows ~300ms sooner |
| Service worker (`/sw.js`) | generated by build | Repeat visits load from cache <100ms |
| Immutable cache for static assets | `_headers` | Browser + Cloudflare edge cache forever |
| HTML short cache + must-revalidate | `_headers` | Updates propagate fast, assets stay cached |

## Form submission flow

```
Browser → POST /api/contact (JSON)
        → Cloudflare Worker (worker/index.js)
        → Brevo Transactional Email API
        → info@ocgt.de inbox
```

The form code in `OCGT_website.html` POSTs to a configurable endpoint
(`FORMSPREE_ENDPOINT` constant). The build script rewrites this constant
to `/api/contact` in the dist HTML only — local dev still falls back to
mailto for safety.

## Local development

No tooling needed. Open `OCGT_website.html` directly, or:
```bash
python3 -m http.server 8000
```
The SPA router handles all 20 routes client-side.

Before deploying, always run `python3 build-prerender.py` to regenerate `dist/`.

## What NOT to do

- ❌ Edit files in `dist/` — they're regenerated on every build.
- ❌ Add `<style>` or `<script>` tags inside individual page `<div id="p-*">` blocks — they will run/repeat on every route.
- ❌ Rename a slug without updating `ROUTES` in `build-prerender.py` AND `sitemap.xml`.
- ❌ Add Tailwind, SvelteKit, Next.js, or any framework. The site is intentionally framework-less.
- ❌ Commit `dist/`. It's a build artifact.

## Future enhancements (in priority order)

1. **Subset custom fonts** if any are added.
2. **Per-route CSS splitting** — only ship the CSS each page needs.
3. **Playwright smoke tests** — automated cross-browser regression catch.
4. **Cloudflare Web Analytics** — privacy-friendly, no cookies, no GDPR banner needed.

(CSS/JS extraction into `assets/site.*.css`/`.js` — content-hashed, cached once across all 20 pages — is already implemented in `build-prerender.py`.)
